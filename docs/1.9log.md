## Critical fixes (must-have)

1. ⏳ **Graph node index safety**
   - `remove_file()` uses `graph.remove_node()` which invalidates petgraph indices.
   - Stale `NodeId` in `file_index`, `name_index`, `id_index` point to wrong nodes after removal.
   - Migrate to `StableGraph` or enforce full-rebuild removal semantics.
   - Goal: prevent silent graph corruption after incremental updates.

2. ⏳ **Legacy C# parser dispatch parity**
   - `get_parser("cs")` returns `None` — no match case in `languages/mod.rs:46-74`.
   - `parser_v2.rs` HAS compiled C# queries. Legacy path silently drops C# files.
   - Goal: align actual behavior with advertised language support.

3. ⏳ **WebSocket sync server hardening**
   - Headless mode binds `0.0.0.0` with zero authentication. Full graph exposed to network.
   - No origin checks, no rate limiting, no token auth.
   - Add `--allowed-origins`, default localhost, optional `ARBOR_TOKEN` env var.
   - Goal: reduce metadata leakage risk in headless/networked use.

## High-priority accuracy fixes

4. ⏳ **Callers/callees edge-kind coverage**
   - `get_callers()` / `get_callees()` filter `EdgeKind::Calls` only.
   - `Imports`, `Extends`, `Implements`, `UsesType` edges all silently dropped.
   - Support relevant non-Calls relationships where appropriate.

5. ⏳ **Dependents traversal correctness**
   - `get_dependents()` uses `queue.pop()` (DFS) but reports hop distance (BFS concept).
   - Nodes 1 hop away can appear at depth 5 if discovered via longer path first.
   - Fix: `VecDeque::pop_front()` for proper BFS.

6. ⏳ **Graph export completeness**
   - `export_graph()` exports nodes but NO edges. `export_edges()` exists. Just not called.
   - One line fix: add `"edges": graph.export_edges()` to export JSON.

7. ⏳ **Language capability response correctness**
   - `handlers.rs:38` hardcodes `vec!["typescript", "rust", "python"]`.
   - Arbor supports 10+ languages. API lies to clients.

8. ⏳ **Builder always creates `Calls` edges**
   - `builder.rs:115-118`: `resolve_edges()` creates `Edge::new(EdgeKind::Calls)` for ALL references.
   - Even imports, type references, extends — all become `Calls`.
   - Edge types completely meaningless in current graph.

9. ⏳ **`export_edges()` panics on corrupted graph**
   - `graph.rs:256,262`: `.unwrap()` on `node_weight()` inside `export_edges()`.
   - If graph has dangling edge references (from BUG #1), this crashes the server.

## Performance targets

10. ⏳ **Indexer parallelization**
    - Files parsed sequentially. Tree-sitter is CPU-bound — perfect for `rayon`.
    - `par_iter().map(parse_file)` then merge results.
    - 2-4x faster indexing on multi-core machines.

11. ⏳ **Search and symbol lookup acceleration**
    - Search index: O(n) false positive filter after n-gram intersection.
    - Symbol table: O(n) suffix matching in `resolve_with_context()`.
    - Fix: reverse `NodeId → name` map + suffix trie.

12. ⏳ **Context token estimation realism**
    - `slice.rs` uses `" ".repeat(lines * 40)` — fake tokens.
    - `handlers.rs:177` uses `(line_end - line_start) * 40 / 4` — different fake heuristic.
    - Two different heuristics for same purpose. Neither reads actual source.
    - Use source-aware estimation instead of fixed-width heuristics.

13. ⏳ **Watch mode efficiency**
    - `watch` command calls `index_directory()` every 2 seconds (full re-walk).
    - `SyncServer` already has proper `notify`-based file watcher with debouncing.
    - Avoid repeated full re-index loops; delegate to sync_server infrastructure.

14. ⏳ **Broadcast sends FULL graph on every file change**
    - `sync_server.rs:666-680` sends ALL nodes + ALL edges on every file change event.
    - Large repo = massive memory/bandwidth waste. DoS vector via rapid file changes.
    - Send delta-only updates (changed file's nodes/edges).

15. ⏳ **Centrality computation O(iterations × nodes × degree)**
    - `ranking.rs` calls `get_callers()` per node per iteration.
    - `get_callers()` does string-based ID lookup via `graph.get_index(&caller.id)`.
    - Pre-compute adjacency lists, use petgraph's native `neighbors_directed()` directly.

## Architecture improvements

16. ⏳ **Split CLI command monolith**
    - `commands.rs` = 82KB, 2472 lines, 16+ commands in one file.
    - Break into `commands/` module directory. One file per command group.

17. ⏳ **Parser-path convergence**
    - `parser.rs` (v1, trait-based) and `parser_v2.rs` (v2, query-based) = two separate systems.
    - C# works in v2 but NOT in v1. Sync server uses v2, CLI uses v1.
    - Unify on v2. Keep v1 as deprecated fallback only.

18. ⏳ **Deduplicate extension lists**
    - File extension lists copy-pasted in 3+ places. Out of sync.
    - `"hxx"` in `supported_extensions()` but missing from sync server list.
    - Single source of truth: `arbor_core::languages::watched_extensions()`.

19. ⏳ **Stale debug comment in parser_v2.rs**
    - Line 115: `// TEMPORARILY DISABLED FOR DEBUGGING` — but Go queries are ENABLED right below.
    - Comment is stale/misleading. Remove it.

26. ⏳ **Config version stuck at "1.0"** — `commands.rs:84-86`. `ensure_arbor_initialized()` writes `"version": "1.0"`.

27. ⏳ **Cache version stuck at "arbor-1.3"** — `store.rs:9`. Bump to `"arbor-1.9"`.

28. ⏳ **MCP input sanitization** — Symbol names from MCP passed directly. Strip path separators and control characters as defense-in-depth.

29. ⏳ **Server `unwrap()` panics** — `sync_server.rs:674,703,746` and `graph.rs:256,262` use `.unwrap()` in production codepaths. Any corrupt data = server crash.
