# Arbor 1.9.0 Release Log

## Release theme

**Performance, correctness, and security hardening** with stronger AI/graph tooling.

---

## Scope definition

This document tracks the **1.9.0 release plan** and implementation priorities.

- ✅ = completed in 1.9.0
- 🔄 = in progress for 1.9.x patch line
- ⏳ = planned, not yet merged

---

## Critical fixes (must-have)

1. ⏳ **Graph node index safety**
   - Migrate from unstable node index assumptions (or enforce full-rebuild removal semantics).
   - Goal: prevent stale index corruption after incremental removals.

2. ⏳ **Legacy C# parser dispatch parity**
   - Ensure C# is not silently dropped in legacy parser path.
   - Goal: align actual behavior with advertised language support.

3. ⏳ **WebSocket sync server hardening**
   - Add auth/origin controls for non-local exposure scenarios.
   - Goal: reduce metadata leakage risk in headless/networked use.

---

## High-priority accuracy fixes

4. ⏳ **Callers/callees edge-kind coverage**
   - Support relevant non-Calls relationships where appropriate.

5. ⏳ **Dependents traversal correctness**
   - Ensure BFS semantics where hop distance is reported.

6. ⏳ **Graph export completeness**
   - Include edges in graph export payload.

7. ⏳ **Language capability response correctness**
   - Remove hardcoded language list from API/handlers.

---

## Performance targets

8. ⏳ **Indexer parallelization**
   - Use multi-core parsing where safe.

9. ⏳ **Search and symbol lookup acceleration**
   - Reduce O(n) fallback scans with reverse indices.

10. ⏳ **Context token estimation realism**
    - Use source-aware estimation instead of fixed-width heuristics.

11. ⏳ **Watch mode efficiency**
    - Avoid repeated full re-index loops; move to file-event deltas.

---

## Architecture improvements

12. ⏳ **Split CLI command monolith**
    - Break large command module into command-group files.

13. ⏳ **Parser-path convergence**
    - Reduce v1/v2 parser divergence and feature mismatch.

14. ⏳ **Edge kind propagation in builder**
    - Preserve relationship semantics through resolution pipeline.

---

## Feature candidates for 1.9.x

15. ⏳ Expanded MCP tools (search, callers/callees, stats, blast radius)
16. ⏳ `arbor dead-code`
17. ⏳ `arbor stats`
18. ⏳ `arbor completions <shell>`
19. ⏳ Cycle detection / SCC reporting
20. ⏳ `arbor diff --against <ref>`
