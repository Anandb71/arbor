# v2.1.0 Design: Agent-Native Deepening

**Date:** 2026-05-05  
**Branch:** `feat/v2.1.0-agent-native`  
**Theme:** Make Arbor the default intelligence layer for AI coding agents

---

## Problem

Current MCP surface (3 tools) forces agents to make wide, expensive calls when they need precise answers. `get_logic_path` returns a full architectural brief when an agent just wants callers of one function. `analyze_impact` returns a blast radius when an agent just wants to know what a file contains. Agents compensate by calling tools repeatedly with slight variations, burning tokens and latency.

Output format is inconsistent ŌĆö some fields are present sometimes, absent others. Agents cannot reliably parse or chain tool calls.

---

## Solution

Two-axis improvement:

1. **Tool Expansion** ŌĆö 6 new surgical MCP tools so agents can ask precise questions
2. **Output Quality** ŌĆö standardized JSON schema across all tools with confidence scores, node roles, and `suggested_next_tool` for agent chaining

---

## New MCP Tools

| Tool | Input | What it returns |
|------|-------|-----------------|
| `list_entry_points` | `path?` | All detected entry points (HTTP routes, main fns, webhooks, background jobs, CLI handlers) with role + file |
| `get_callers` | `symbol`, `depth?` (default 1) | Direct callers of symbol, one hop upstream, with confidence |
| `get_callees` | `symbol`, `depth?` (default 1) | Direct callees of symbol, one hop downstream |
| `search_symbols` | `query`, `kind?`, `limit?` | Fuzzy name search across graph ŌĆö returns ranked list with file + kind |
| `get_file_graph` | `file_path` | All nodes + edges within a single file |
| `get_node_detail` | `symbol` | Full detail: file, line range, kind, role, confidence score, centrality rank |

Existing tools (`get_logic_path`, `find_path`, `analyze_impact`) kept unchanged for backwards compatibility.

---

## Output Quality Standard

Every tool response, new and existing, will conform to this envelope:

```json
{
  "ok": true,
  "tool": "<tool_name>",
  "arbor_version": "2.1.0",
  "data": { ... },
  "meta": {
    "confidence": 0.92,
    "suggested_next_tool": "get_callers",
    "suggested_next_args": { "symbol": "checkout" },
    "node_count": 4,
    "truncated": false
  }
}
```

**Node objects** in all responses gain two new fields:
- `role`: `"entry_point" | "utility" | "internal" | "test_helper" | "unknown"`
- `confidence`: `0.0ŌĆō1.0` (edge confidence from `ConfidenceExplanation`)

**Causality chain** added to `get_logic_path` and `analyze_impact` ŌĆö ordered list of `(caller ŌåÆ callee, reason)` pairs explaining why A affects B.

1. `cargo clippy --workspace --all-targets --all-features -- -D warnings` ŌĆö fix all warnings
2. `cargo test --workspace` ŌĆö all tests must pass

No new feature ships until audit is clean.

## Architecture Impact

Changes confined to:
- `crates/arbor-mcp/src/` ŌĆö new tool handlers
- `crates/arbor-graph/src/` ŌĆö expose new query methods needed by tools
- `crates/arbor-cli/src/commands.rs` ŌĆö no changes needed (MCP is separate binary)

## Success Criteria

- Claude Code / Cursor can answer "what calls `checkout`?" with one `get_callers` call

- All tool responses parse without conditional field checks
- `cargo clippy -D warnings` clean
- All existing tests pass + new tests for each new tool
- CI green on Ubuntu + macOS + Windows
